1. 复制文章 API 链接

打开任意文章详情页，点击 API 数据 按钮。按钮会自动复制当前文章的 API 链接，不会打开新页面。

2. 使用 VIP 账号请求

API 读取依赖当前登录态。请先在浏览器登录 VIP 账号，或在你的请求工具里携带同一账号的会话 Cookie。

3. 控制请求次数

每个 VIP 账号对每篇文章在 {{ api_window_days }} 天内最多请求 {{ api_limit }} 次。管理员不受次数限制。

请求示例

复制后可直接替换 URL

浏览器或 Cookie 请求

fetch("{{ sample_api_url|default:'https://example.com/blog/article-slug/api/' }}", {
  method: "GET",
  credentials: "include",
  headers: {
    "Accept": "application/json"
  }
})
  .then(response => response.json())
  .then(data => console.log(data.article.title));

Python 分析脚本

import requests

url = "{{ sample_api_url|default:'https://example.com/blog/article-slug/api/' }}"
cookies = {"sessionid": "你的登录会话"}

response = requests.get(url, cookies=cookies, headers={"Accept": "application/json"})
response.raise_for_status()

payload = response.json()
article = payload["article"]

print(article["title"])
print(article["content"])
print(article["stats"])

返回数据

适合 AI 分析和结构化处理

成功返回时，JSON 中会包含以下核心字段：

  • usage：当前请求窗口、次数上限、已用次数、剩余次数、重置时间。
  • article.title：文章标题。
  • article.author：作者账号、展示名和主页链接。
  • article.content：原始正文，保留 Markdown 或富文本内容。
  • article.content_html：服务端渲染并清洗后的 HTML。
  • article.stats：阅读量、API 请求总次数、点赞、收藏、评论、字数和阅读时长。
  • article.comments：评论和回复列表，包含作者、内容、置顶状态和时间。

状态码

  • 200：请求成功。
  • 401：未登录。
  • 403：账号不是 VIP 或管理员。
  • 404：文章不存在，或当前账号不可见。
  • 429：{{ api_window_days }} 天内 {{ api_limit }} 次请求额度已用完。
