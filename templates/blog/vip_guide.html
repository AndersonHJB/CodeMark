获取 Cookies

用于脚本或请求工具保持 VIP 登录态

备用方法一：浏览器开发者工具

  1. 先在网站登录你的 VIP 账号。
  2. 打开开发者工具，进入 Application 或 应用 面板。
  3. 在 Cookies 中选择当前站点域名。
  4. 复制 sessionid 的值。这个值就是服务端识别登录态的关键 Cookie。
Cookie: sessionid=你的-sessionid-值

备用方法二：复制为 cURL

  1. 登录 VIP 账号后，在浏览器打开任意文章 API 链接。
  2. 打开开发者工具的 Network 面板，选中这次 API 请求。
  3. 右键选择 Copy as cURL。复制出的命令会自动带上 Cookie。
curl "{{ sample_api_url|default:'https://example.com/blog/article-slug/api/' }}" \
  -H "Accept: application/json" \
  -H "Cookie: sessionid=你的-sessionid-值"

备用方法三：Python requests

import requests

url = "{{ sample_api_url|default:'https://example.com/blog/article-slug/api/' }}"
cookies = {{ current_cookie_python }}

response = requests.get(url, cookies=cookies, headers={"Accept": "application/json"})
response.raise_for_status()

payload = response.json()
print(payload["article"]["title"])

Cookie 相当于登录凭证，只应保存在自己的本地环境或服务器密钥管理中，不要放进公开仓库、公开网页或分享给他人。

返回数据

适合 AI 分析和结构化处理

成功返回时，JSON 中会包含以下核心字段：

  • usage：当前请求窗口、次数上限、已用次数、剩余次数、重置时间。
  • article.title：文章标题。
  • article.author：作者账号、展示名和主页链接。
  • article.content：原始正文，保留 Markdown 或富文本内容。
  • article.content_html：服务端渲染并清洗后的 HTML。
  • article.stats：阅读量、API 请求总次数、点赞、收藏、评论、字数和阅读时长。
  • article.comments：评论和回复列表，包含作者、内容、置顶状态和时间。

状态码

  • 200：请求成功。
  • 401：未登录。
  • 403：账号不是 VIP 或管理员。
  • 404：文章不存在，或当前账号不可见。
  • 429：{{ api_window_days }} 天内 {{ api_limit }} 次请求额度已用完。
